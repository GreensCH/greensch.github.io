进一步理解理解RISCV的内存地址

pmem是host中使用uint8_t类型的数组，用于模拟(充当)NEMU的内存

在guest(NEMU)的视角来看，pmem是一块物理内存对应地址为paddr，paddr=随机数1, paddr=随机数2, paddr=随机数3… 对应的pmem[0], pmem[1], pmem[2]…的pmem真实地址（也就是host执行NEMU时定义pmem数组后，这些数组内不同变量的地址）

从guset执行程序的视角来看，guest执行程序时guset分配的地址是vaddr，即0x80000000等

通过guest_to_host()，使得guset到pmem，guset起始地址对应pmem[0]，RISCV起始地址为0x80000000则paddr = 0x80000000最终会访问pmem[0]

一些需要注意的数据：

  • pmem大小/nemu内存大小：128MB
  • RICSV开始地址为0x80000000，在guest_to_host函数中特意减去了改开始地址，因此guset中无法访问比0x80000000小的地址值，否则会报错
  • CONFIG_MBASE：代表起始值，在RISCV64中值为0x80000000
  • CONFIG_MSIZE = 0x8000000
  • RESET_VECTOR = (CONFIG_MBASE + CONFIG_PC_RESET_OFFSET)
  • CONFIG_PC_RESET_OFFSET = 0x0
  • static uint8_t pmem[CONFIG_MSIZE] PG_ALIGN = {};
