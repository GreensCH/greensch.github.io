Questasim in Linux

使用linux上10.7版本的Questasim对仿真强制优化，从而看不见模块内部变量，无法进行仿真时修改

解决方法：

  • vsim -voptargs=+acc work.xxx
    • work.xxx xxx是仿真的文件名
    • -voptargs：vsim optimision arguments =+ access
      • access：表示当前所有信号都可看到

其他基本命令

仿真vsim

几个示例：

  • vsim -novopt -solvefaildebug -sv seed random work.tb1
  • vsim -novopt -classdebug -solvefaildebug -sv_seed 0 +TESTNAME=mcdf_data_consistence_basic_test -l mcdf_data_consistence_basic_test_sim.log work.tb
  • vsim -i -novopt -classdebug -solvefaildebug -coverage -coverstore ./mti_covdb -testname mcdf_full_random_test -sv_seed random +TESTNAME=mcdf_full_random_test -l mcdf_full_random_test.log work.tb

参数：

  • -sv_seed [num]，传递随机种子，如果值为random则值表示随机，如果为固定值则会产生相同的数据

  • -solvefaildebug，仿真失败后出现调试信息（用于看随机化失败原因）使用示例：

  • -classdebug：这是为了提供更多对SV类调试功能（如View->ClassBrowse功能）
  • +TESTNAME=mcdf_data_consistence_basic_test：我们在tb.sv中通过调用系统函数得到的参数TESTNAME，用来指定仿真选择的调试
  • -l mcdf_data_consistence_basic_test_sim.log：这是让仿真的记录保存在特定的测试文件名称中
  • -coverage：会在仿真是产生代码覆盖率数据，功能覆盖率则默认会生成，与此选项无关
  • coverstore COVERAGE_STOREAGE_PATH：用来在仿真结束时，生成覆盖率数据并且存储到COVERAGE_STOREAGE_PATH路径
  • -testname TESTNAME：这个选项是你需要添加本次仿真的test名称，可以使用同+TESTNAME选项一样的test名称。这样在仿真结束后，将在COVERAGE_STORAGE_PATH下产生一个覆盖率数据文件{TESTNAME}_{SV_SEED}.data。由于仿真时传入的种子是随机值，因此每次提交测试，在测试结束后都将产生一个独一无二的覆盖率数据

结束仿真：quit -sim

其他仿真操作

compile by order：自动按顺序编译，可以自动区分设计与验证文件

我们的手动顺序如下

  • 参数文件param_def没有必要编译
  • 先编译设计的部分，其中顶层设计mcdf要最后一个编译，编译要有层次感
  • 类似设计部分，再对验证部分进行编译
  • 最后编译顶层tb

保存波形：

  • log -r /*：保存全部仿真波形
    • -r表示递归
    • /*表示层级下所有信号
